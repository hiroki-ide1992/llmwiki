# LLM Wiki — エージェント設定ファイル

このファイルはLLMエージェントへの指示書です。ウィキの構造・規約・ワークフローを定義します。

---

## ディレクトリ構成

```
raw/        # ソース文書（読み取り専用・絶対に編集しない）
wiki/       # LLMが生成・管理するウィキページ群
schema/     # このファイルを含む設定ドキュメント
```

### 各レイヤーの役割

- **raw/** — 収集した一次情報源（記事・論文・メモなど）。不変。LLMは読むが書き込まない。
- **wiki/** — LLMが作成・更新するマークダウンファイル群。エンティティページ・概念ページ・要約・比較などを含む。
- **schema/** — このAGENT.mdを含む設定ファイル。ウィキのルールとワークフローを定義する。

---

## ウィキの構造規約

### ページ種別

| 種別 | 用途 | 命名例 |
|------|------|--------|
| エンティティページ | 人物・組織・製品など固有の概念 | `nginx.md`, `CloudFlare.md` |
| 概念ページ | 技術・理論・アイデアなど | `リバースプロキシ.md`, `負荷分散.md` |
| ソースページ | rawの各ファイルの要約 | `source_リバースプロキシとは.md` |
| 比較ページ | 複数の概念・製品の対比 | `nginx_vs_Apache.md` |
| index.md | ウィキ全体の目次（特殊ファイル） | — |
| log.md | 操作ログ（追記専用・特殊ファイル） | — |

### フォーマット規約

- すべてのページはMarkdown形式
- ページ先頭にYAMLフロントマター（任意だが推奨）:
  ```yaml
  ---
  tags: [概念, ネットワーク]
  sources: [リバースプロキシとは.md]
  updated: 2026-06-28
  ---
  ```
- 関連ページへのリンクは `[[ページ名]]` 形式（Obsidian互換）
- 矛盾・未解決の疑問点は `> ⚠️ 矛盾: ...` として明示する

## 特殊ファイル

### wiki/index.md

- ウィキ内の全ページをカテゴリ別に一覧化する
- 各エントリにページへのリンクと1行の説明を含める
- ソース取り込みのたびに更新する
- クエリ回答時はまずindex.mdを読んでから関連ページに深掘りする

フォーマット例:
```markdown
## 概念
- [[リバースプロキシ]] — クライアントとサーバーの間に立つプロキシの一形態
- [[負荷分散]] — 複数サーバーへリクエストを振り分ける仕組み

## エンティティ
- [[nginx]] — 高性能なWebサーバー兼リバースプロキシ
```

### wiki/log.md

- 追記専用。既存エントリは編集しない
- 各エントリのプレフィックス形式: `## [YYYY-MM-DD] 操作種別 | タイトル`
- 操作種別: `ingest`（取り込み）、`query`（質問）、`lint`（健全性チェック）

例:
```markdown
## [2026-06-28] ingest | リバースプロキシとは.md
要約ページを作成。nginx・Apache・CDNのページを更新。
```

## オペレーション

### 1. Ingest（ソース取り込み）

新しいソースが `raw/` に追加されたとき:

1. ソースファイルを読む
2. 重要なポイントについてユーザーと議論する（必要に応じて）
3. `wiki/source_<ファイル名>.md` に要約ページを作成する
4. 関連するエンティティ・概念ページを更新または新規作成する（1ソースで10〜15ページ程度触ることがある）
5. 新情報が既存の記述と矛盾する場合は明示的にフラグを立てる
6. `wiki/index.md` を更新する
7. `wiki/log.md` にエントリを追記する

### 2. Query（質問への回答）

1. `wiki/index.md` を読んで関連ページを特定する
2. 該当ページを読んでウィキ内の情報を統合する
3. 引用付きで回答する（例: `([[リバースプロキシ]]より)`）
4. 価値ある回答はウィキに新しいページとして保存することを提案する

### 3. Lint（健全性チェック）

定期的に以下を確認する:

- ページ間の矛盾・不一致
- より新しいソースで覆された古い記述
- リンク先のないorphanページ
- 言及はされているが独自ページのない重要概念
- 欠落しているクロスリファレンス
- 調査が必要なデータギャップ

## 注意事項

- `raw/` 内のファイルは**絶対に編集しない**。読み取り専用。
- `wiki/log.md` は**追記のみ**。過去のエントリを修正しない。
- ページを削除する場合はlog.mdに記録してから削除する。
- 不確かな情報には `(要確認)` を付ける。
- ハルシネーション防止のため、rawソースに根拠のない記述はウィキに書き込まない。
