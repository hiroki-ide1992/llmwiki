# Throughput（スループット）

一定時間あたりにシステムが処理できる量を表す性能指標。

## 定義

> **Throughput** = システムがどれだけの量をさばけるかを表す性能指標

## 代表的な単位

| 単位 | 意味 |
|------|------|
| RPS / req/s | 1秒あたりのリクエスト数 |
| TPS | 1秒あたりのトランザクション数 |
| MB/s・GB/s | 1秒あたりのデータ転送量 |
| IOPS | 1秒あたりのI/O回数 |

## Latency との違い

| 用語 | 問い | 比喩 |
|------|------|------|
| Throughput | 大量にさばけるか？ | レジが1時間に何人さばけるか |
| Latency | 1件にどれだけかかるか？ | 1人の会計が終わるまでの秒数 |

Throughput が頭打ちになると待ち行列が発生し、**Latency も連動して悪化**する。

## インフラでのボトルネック

API サーバーなどで Throughput が伸びない原因:

- CPU 使用率の高騰
- DB 接続数の枯渇
- ネットワーク帯域の不足
- OpenSearch の search queue 詰まり
- ECS タスク数の不足
- ALB 配下サーバーの処理能力不足

インフラ監視では CPU 使用率・Latency と合わせて確認し、**どこがボトルネックで処理量が伸びないか**を特定する。
