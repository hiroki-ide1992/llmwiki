# ソース要約: メモリのスワップとは

## 概要

RAM が不足したときに、ストレージの一部を一時的な退避先として使うスワップの仕組みと、監視時に何を見るべきかを説明したドキュメント。

## 主要なポイント

### スワップの基本

- スワップは RAM 不足時に SSD や HDD の一部を一時メモリとして使う仕組み
- すぐに使わないデータを RAM からスワップ領域へ退避し、必要時に RAM へ戻す
- RAM が増えたり高速化したりするわけではなく、あくまで保険として機能する

### メリット

- メモリ不足ですぐにアプリや OS が終了するのを防ぎやすい
- 使われていないデータを退避させ、限られた RAM を有効活用できる

### デメリット

- ストレージは RAM より大幅に遅いため、頻繁なスワップは性能低下を招く
- スワップが多発する状態はスラッシングと呼ばれる

### 監視で見るべき点

- スワップ使用量そのもの
- スワップへの書き出し・読み戻しが継続しているか
- 使用量が残っていても、継続的な入出力がなければ必ずしも異常ではない

### Linux での確認例

- free -h
- swapon --show

# メモリのスワップ

メモリのスワップは、RAM が不足したときにストレージの一部を一時的な退避先として使う仕組み。

## 仕組み

- すぐに使わないデータを RAM からスワップ領域へ移す
- 再び必要になったデータは RAM へ戻す
- 退避先は SSD や HDD のため、RAM より大幅に低速

## 役割

スワップはメモリ不足時の安全弁として働き、即座のプロセス終了やシステム不安定化を避けやすくする。

## 性能上の注意

- スワップ使用自体は即異常ではない
- 頻繁なスワップイン・スワップアウトは性能劣化の兆候
- 継続的に発生するとスラッシングにつながる

## 監視観点

- Swap 使用量
- スワップイン・スワップアウトが継続しているか
- メモリ不足が断続的か継続的か

## よくある誤解

スワップを設定しても RAM の容量や速度が増えるわけではない。あくまで不足時の退避先。

## Linux での確認

- free -h
- swapon --show
