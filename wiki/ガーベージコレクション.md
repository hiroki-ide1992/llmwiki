# ガーベージコレクション

ガーベージコレクション（GC）は、実行環境（主に JVM）が不要になったメモリ領域を自動で回収する仕組み。

## インフラ運用での位置づけ

GC は必要な処理であり、発生自体は正常。問題になるのは「頻度」と「停止時間」が増えたとき。

- 適度な GC: 通常運用
- 頻繁または長時間の GC: 性能劣化や障害の兆候

## 典型的な影響

- リクエスト処理の遅延
- タイムアウト増加
- 5xx 増加
- CPU 使用率上昇
- 検索系ワークロードでの遅延拡大

## 監視観点

- JVM Memory Pressure
- GC 回数
- GC 時間
- CPU 使用率
- Search latency
- 5xx / timeout

## 悪化シグナル

- メモリ圧迫が継続する
- GC が連続して走る
- 1 回あたりの GC 時間が伸びる
- 同時に [[Latency]] が悪化し、処理待ちが増える

## 主な原因

- Heap サイズ不足
- 重いクエリ・集計
- キャッシュや fielddata の肥大
- shard 設計の不均衡
- ノード資源不足
- indexing 負荷過多

## 運用上の要点

GC は「悪」ではなく、負荷とメモリ設計の結果として現れる指標。単体で判断せず、[[Latency]] や [[Throughput]] と合わせて評価する。
