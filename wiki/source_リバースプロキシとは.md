# ソース要約: リバースプロキシとは

## 概要

リバースプロキシ（nginx を例に）がどのように動作し、なぜ CORS 制限を回避できるように見えるかを平易な日本語で解説したドキュメント。

## 主要なポイント

### リバースプロキシの基本構造

```
ブラウザ → http://localhost:8080 → nginx → https://example.com
```

ブラウザから見えるアクセス先は `localhost:8080`（nginx）のみ。nginx が裏側で外部サイトへアクセスしてレスポンスを返す。

### CORS がかからない理由

- CORS はブラウザが課す制限であり、**サーバー間通信には適用されない**
- ブラウザ視点では「localhost にアクセスしているだけ」なのでオリジン違反が発生しない
- nginx → 外部サイト の通信はサーバー間通信のため CORS 対象外

> 正確には「CORSを突破」しているのではなく、ブラウザが CORS 判定する対象を nginx に変えているだけ

### CORS が守っているもの

悪意あるWebページがユーザーのブラウザを経由して、別サイトの情報を無断取得することを防ぐ仕組み。

### スクレイピングとの区別

- CORS 回避できること ≠ スクレイピングしてよいこと
- 利用規約・robots.txt・アクセス頻度・著作権などに注意が必要
